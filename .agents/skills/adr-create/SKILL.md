# Create ADR

Источник правил: `docs/process/PROCESS.md`.

## Inputs
- Context (почему возник вопрос).
- Decision (что выбрали).
- Consequences (что это меняет/какие риски).

## Procedure
1. Убедись, что ADR действительно нужен (есть несколько вариантов и решение влияет на будущее).
2. Создай новый файл в `spec/adr/` со следующим порядковым номером и кратким слагом.
3. Заполни секции `Context`, `Decision`, `Consequences` по формату из процесса.
4. Добавь ADR в текущий PR или открой отдельный PR с трассировкой на issue (по процессу).

## Output Contract
Верни:
- `Result`: `success` или `fail`.
- `Artifacts`: путь к ADR файлу.
- `Next`: например `pr-open-from-issue`.
