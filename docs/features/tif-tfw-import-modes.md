# TIF+TFW: режимы импорта и выбор пользователем

Краткая памятка по пунктам меню `Импорт -> Импорт TIF` для режима `TIF + TFW`.

## Что общее для всех режимов `TIF + TFW`
- Пользователь выбирает только `*.tif/*.tiff`.
- Одноименный `*.tfw` ищется автоматически в той же папке.
- `TFW` должен содержать 6 строк в порядке `A, D, B, E, C, F`.
- Поворот в `TFW` (`B != 0` или `D != 0`) в MVP не поддерживается.

## 1) `TIF + TFW (градусы)`
- Интерпретация: значения `TFW` считаются в градусах (`EPSG:4326`).
- Когда выбирать:
  - исходные координаты уже в широте/долготе (WGS84);
  - в `TFW` значения похожи на географические координаты (например, `55.x`, `37.x`).
- Отличие от других: не требует метра-проекций и не просит UTM-зону.

## 2) `TIF + TFW (WebMercator, EPSG:3857)`
- Интерпретация: значения `TFW` в метрах Web Mercator (`EPSG:3857`), далее преобразование в `EPSG:4326` для отображения.
- Когда выбирать:
  - растр/привязка получены из веб-карт или тайловых сервисов;
  - координаты в `TFW` большие по модулю, типичные для метров 3857.
- Отличие от других: фиксированная метрическая проекция (без выбора зоны).

## 3) `TIF + TFW (UTM зона)`
- Интерпретация: значения `TFW` в метрах UTM, с ручным выбором зоны и полушария.
- Перед выбором файлов открывается окно UTM-параметров:
  - номер зоны: только `1..60`;
  - полушарие: `N` или `S`.
- Когда выбирать:
  - данные пришли в UTM (геодезия, GNSS, инженерные съемки);
  - у источника явно указаны UTM-зона и полушарие.
- Отличие от других: требует дополнительного ввода зоны/полушария.

## Как быстро выбрать нужный режим
1. Если координаты как широта/долгота (`lat/lon`) -> `градусы`.
2. Если координаты в метрах веб-карт (`EPSG:3857`) -> `WebMercator`.
3. Если в метрах UTM и известна зона -> `UTM зона`.
