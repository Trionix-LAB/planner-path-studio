# T-70: Refactor MapWorkspace boundaries

**Issue:** #70
**Branch:** `chore/70-mapworkspace-refactor`
**Spec:** n/a (structural refactor, behaviour unchanged)

## Why

`src/pages/MapWorkspace.tsx` (≈4 500 строк) совмещает mission lifecycle,
autosave, telemetry runtime, overlay management, UI state и управление
объектами в одном модуле. Это повышает стоимость изменений и риск
регрессий (A-0067, P1). Цель — разбить его на специализированные хуки
с явными интерфейсами, оставив `MapWorkspace` преимущественно
композицией.

## Goal

- `MapWorkspace` содержит преимущественно JSX-разметку и прокидывание props.
- Выделенные хуки/модули имеют чёткие входы/выходы и могут тестироваться отдельно.
- Поведение экрана карты не меняется (`docs/screens.md`).

## Proposed module boundaries

| Модуль | Область ответственности | Примерное размещение |
|---|---|---|
| `useMissionLifecycle` | load/create/open/save миссии, draft, WAL | `src/features/mission/hooks/` |
| `useAutosave` | debounced autosave, статус `saved/saving/error` | `src/features/mission/hooks/` |
| `useTelemetryRuntime` | провайдеры (Zima/GNSS/COM/simulation), connection state, recording | `src/features/mission/hooks/` |
| `useRasterOverlays` | CRUD растровых слоёв, декодирование TIFF, object URL | `src/features/map/rasterOverlays/` |
| `useVectorOverlays` | CRUD векторных слоёв, парсинг DXF, кэш | `src/features/map/vectorOverlays/` |
| `useMapUiState` | activeTool, selectedObjectId/AgentId, panels, mapView, centerRequest | `src/features/map/hooks/` |
| `useAppSettings` | загрузка/сохранение настроек приложения, coordPrecision, styles, grid | `src/features/settings/hooks/` |
