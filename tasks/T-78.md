# T-78: Align PR body enforcement workflow with repo template

**Issue:** #78
**Spec:** docs/process/AGENT_WORKFLOWS.md (PR Body contract)

## Why
* GitHub Action `.github/workflows/enforce-pr-body-why-plan-spec-fixes.yml` currently requires `Plan: tasks/T-XXXX.md`, but the repository workflow and PR template require `Tasks Log: tasks/T-XXXX.md`.
* This mismatch causes PR checks to fail even when PRs follow the documented template.

## Goal
* Make the PR body enforcement workflow validate `Tasks Log:` instead of `Plan:`.
* Keep error messages/examples consistent with the documented PR body format.

## Plan
* [ ] Update `.github/workflows/enforce-pr-body-why-plan-spec-fixes.yml`:
  - rename references from Plan -> Tasks Log
  - update regex and failure output example
  - update workflow name/step name for clarity

## Acceptance
* [ ] PR body check passes when PR description contains:
  - `Fixes #...`
  - `Spec: ...`
  - `Tasks Log: tasks/T-XXXX.md`
  - `Why: ...`
* [ ] Failure message shows correct field name and example.
