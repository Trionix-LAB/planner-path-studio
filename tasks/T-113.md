# T-113: Асинхронный TIFF-декодер в Electron main process

**Issue:** local-fix (без GitHub Issue)
**Spec:** Без изменения требований (внутренний perf/runtime fix)

## Why
- Синхронный `decodeTiffToPng` выполняется в `electron` main process и блокирует event loop на больших `TIF/GeoTIFF`.
- В результате при подгрузке растров заметны лаги интерфейса.

## Goal
- Перевести встроенный TIFF-декодер на асинхронное выполнение через `worker_threads`, сохранив текущую логику fallback.
- Убрать блокировку main process во время декодирования.

## Plan
- [x] Добавить async-обертку `decodeTiffToPngAsync` в `electron/tiff-decoder.cjs` с выполнением в worker.
- [x] Обновить `electron/main.cjs` и использовать `await decodeTiffToPngAsync(inputBuffer)` в IPC-обработчике.
- [x] Проверить локально на `tools/test_tifs/*.tif`, что декодирование работает.

## Acceptance
- [x] IPC `planner:raster:convertTiffBase64ToPngBase64` не выполняет встроенный TIFF-декод синхронно в main thread.
- [x] Декодирование возвращает PNG base64 как и раньше при успешном built-in decode.
