# T-101: Toggle visibility for agent/base station tracks

**Issue:** #101
**Spec:** New requirement

## Context
Track rendering is currently controlled by a single layer flag:
* `layers.track` (Settings -> "Видимость слоев" and LeftPanel) toggles *all* track polylines.
* Map rendering: `src/components/map/MapCanvas.tsx` renders `trackSegments` when `layers.track` is true.
* Track segments come from `MapWorkspace`:
  - `buildTrackSegments(trackPointsByTrackId)` -> segments per `trackId`
  - `MapWorkspace` maps segments to `{ points, color }` using track meta color.

Base station now writes tracks through the same pipeline as agents using `agent_id='base-station'`.

Requirement: add separate controls to hide/show:
* agent tracks
* base station track
while keeping current markers/positions visible.

## Goal
* Allow hiding/showing tracks **independently per track**:
  - each agent track can be hidden separately
  - each base station track can be hidden separately
* For the currently selected agent / base station, support both:
  - hide/show individual track items (per-track eye)
  - hide/show ALL tracks for the selected entity (one action)
* Persist hidden/shown state between sessions.
* Preserve backward compatibility with existing `layers.track` master toggle (hide all tracks).

## Recommended design (minimal disruption)
* Keep `layers.track` as a master switch (existing UI behavior).
* Introduce a per-track hidden set stored in mission UI:
  - `mission.ui.hidden_track_ids?: string[]` (source of truth)
  - normalize to a `Set<string>` in runtime for filtering
* Rendering rules:
  - if `layers.track === false`: render no tracks
  - else: render track segments only for trackIds NOT present in `hidden_track_ids`

Rationale:
* Per-track visibility is mission-specific (track ids belong to a mission), so persisting in mission UI state is the safest and most direct approach.

## Implementation plan (agent)
1) Spec
* Update `spec/spec.md` with a new requirement (R-0xx) describing:
  - new toggles
  - defaults
  - persistence
  - relationship with existing `layers.track` (master)

2) Mission UI type + persistence
* Update `MissionUiState` in `src/features/mission/model/types.ts`:
  - add `hidden_track_ids?: string[]`
* Ensure `buildMissionBundle` / persistence path in `src/pages/MapWorkspace.tsx` includes the field.
* When opening a mission, hydrate `hidden_track_ids` into local state (if you keep local state) or use mission ui directly.

3) MapWorkspace filtering
* In `src/pages/MapWorkspace.tsx`:
  - build `hiddenTrackIds` as `Set<string>` from `missionDocument.ui?.hidden_track_ids ?? []`.
  - when producing `trackSegments`, include `trackId` (so MapCanvas can key/filter) OR filter before passing to MapCanvas.
  - update the mission snapshot builder to persist `hidden_track_ids` when it changes.

4) UI (Right panel per-track toggle)
* In `src/components/map/RightPanel.tsx` "Tracks" section (when an agent or base station is selected):
  - render the list of tracks for the selected entity (already exists)
  - for each track row add an eye icon button:
    - visible: `Eye`
    - hidden: `EyeOff`
  - clicking toggles that specific `track.id` in `hidden_track_ids`.
  - IMPORTANT: this must work for both diver tracks and base station tracks.
  - keep current positions/markers visible (tracks-only).

* Add an entity-level control (in the same Tracks header area) to hide/show ALL tracks for the selected entity:
  - If at least one visible track exists for that entity: action hides all (adds all ids to `hidden_track_ids`).
  - If all tracks are hidden for that entity: action shows all (removes those ids from `hidden_track_ids`).
  - Use an eye icon on the header as well (or Eye/EyeOff pair) and a tooltip that clarifies the action.

5) UI (Settings)
* Keep the existing global "Трек" layer checkbox (master) as-is.
* Do NOT add per-track controls to Settings (they belong in the Tracks list UI).

6) Rendering
* Update `trackSegments` shape to include which entity it belongs to:
  - in `MapWorkspace` produce `{ trackId, points, color }` per segment
  - update `MapCanvas` prop typing and rendering to use `trackId` for stable keys.
  - filter segments using `hidden_track_ids` before passing to `MapCanvas` (recommended).

7) Tests
* Add unit test for filtering logic (recommended):
  - Given multiple trackIds, ensure hidden ids are excluded when `layers.track` is enabled.
* Add UI tests for RightPanel:
  - toggling the eye button hides/shows the specific track
  - entity-level eye control hides/shows all tracks for selected agent/base station
  - verify persistence: simulate remount/open mission state containing `hidden_track_ids`.

8) Verify
* `npm run typecheck`
* `npm run lint`
* `npm run test`

## Acceptance checklist
* Two track visibility toggles exist and default to enabled.
* Agent tracks and base station track can be hidden independently.
* Settings persist between sessions via existing settings/mission UI flow.
* No regressions for existing `layers.track` master toggle.
* Tests cover behavior.
