# T-102: Импорт DXF/DWG на карту (координаты UTM из файла)

**Issue:** #102
**Spec:** New requirement (R-062)

## Context

* Карта уже поддерживает растровые наложения (GeoTIFF / TFW) через `raster_overlays` в `MissionUiState`.
* Векторных overlay-слоёв нет; DXF/DWG не поддерживаются.
* Внутренний CRS карты — WGS84 (EPSG:4326); вся геометрия хранится в этом CRS.
* DXF/DWG-файлы с инженерными чертежами задают координаты в UTM (метры, зона + полушарие).
  - В MVP: ручная геопривязка (GCP/аффинные матрицы) не требуется — достаточно конвертации UTM → WGS84.
* Уже есть `src/features/geo/crs.ts`; UTM <-> WGS84 конвертацию следует добавить туда же
  (либо отдельным модулем `src/features/geo/utm.ts`).
* DWG — бинарный формат; поддержка должна выполняться нативным DWG-парсером (без `DWG -> DXF` конвертации).
* При импорте должен учитываться только model space (геометрия рисунка): рамка/штамп из layout/paper space не должны попадать в слой и влиять на bounds/центр.
* Для ускорения повторного открытия миссии нужен персистентный кэш распарсенной CAD-геометрии в папке миссии (по аналогии с хранением растровых артефактов в `overlays/`).

## Goal

* Импорт DXF/DWG-файла как пользовательского векторного overlay-слоя поверх карты.
* Геометрия из DXF/DWG позиционируется на карте через конвертацию UTM → WGS84.
* UI управления слоем: visible / opacity / z-order (аналогично raster overlays).
* Слой сохраняется в папке миссии и восстанавливается при повторном открытии.
* Метаданные слоя хранятся в `mission.ui.vector_overlays`.
* Для слоя сохраняется `cache_file` с WGS84-геометрией/bounds для `cache-first` загрузки при открытии миссии.

## Implementation plan

- [x] **1. Spec** — добавить/обновить `R-062` в `spec/spec.md`:
  - внутренний CRS: WGS84 (EPSG:4326)
  - поддерживаемый входной CRS для DWG/DXF: UTM (зона 1–60, полушарие N/S)
  - явная последовательность в UI: `ToolBar -> Импортировать -> Импортировать DWG/DXF -> DWG/DXF + UTM -> Окно выбора зоны UTM`
  - правило выбора UTM-параметров: из файла (заголовок `$EXTMIN`/`$EXTMAX` или CRS-аннотация), иначе — диалог
  - единицы DXF: метры (MEASUREMENT=1); при несовпадении — ошибка или предупреждение с настройкой масштаба
  - модель хранения: файл копируется в `<mission_dir>/overlays/<id>.dxf`; метаданные — в `mission.ui.vector_overlays[]`

- [x] **2. DXF-парсер (MVP)** — реализован встроенный parser без внешней зависимости:
  - добавлен `src/features/map/dxfOverlay/parseDxf.ts` (поддержка `LINE`, `LWPOLYLINE`, `POLYLINE`, `POINT`, `INSERT`);
  - совместим с web/electron, без нативных зависимостей;
  - бинарный контент отклоняется как невалидный текстовый DXF.

- [x] **3. UTM → WGS84 конвертация** — реализован `src/features/geo/utm.ts`:
  - `utmToWgs84({ easting, northing, zone, hemisphere }): { lat, lon }`
  - `wgs84ToUtm({ lat, lon }): { easting, northing, zone, hemisphere }`
  - pure, unit-testable, без side-effects

- [x] **4. DXF reader** — реализован `src/features/map/dxfOverlay/parseDxf.ts`:
  - принимает строку (содержимое `.dxf`)
  - возвращает `DxfFeatureCollection` (GeoJSON-подобная структура, геометрия в WGS84)
  - поддерживаемые примитивы MVP: `LINE`, `LWPOLYLINE`, `POLYLINE`, `POINT`, `INSERT` (block reference — базовый)
  - конвертирует каждую точку UTM → WGS84 (зона/полушарие передаются параметром)
  - импортирует только model-space сущности (`group code 67 != 1`), чтобы рамка листа/layout не влияли на геометрию и привязку слоя
  - валидация единиц DXF (`UNITS_NOT_METERS`) вынесена в follow-up (вне минимального контракта R-062)

- [x] **5. Типы и хранение** — `MissionUiState` расширен `vector_overlays`:
  ```ts
  vector_overlays?: Array<{
    id: string;
    name: string;
    file: string;          // относительный путь в папке миссии
    type: 'dxf' | 'dwg';
    file_encoding?: 'utf8' | 'base64';
    utm_zone: number;      // 1..60
    utm_hemisphere: 'N' | 'S';
    opacity: number;       // 0..1
    visible: boolean;
    z_index: number;
  }>;
  ```
  - обновлены `buildMissionBundle`/`persist`/`hydrate` пути в `MapWorkspace.tsx`

- [x] **6. Импорт UI** — добавлен пункт меню и UTM-диалог в `TopToolbar`:
  - путь: `Импорт -> Импортировать DWG/DXF -> DWG/DXF + UTM`;
  - выбор зоны `1..60` и полушария `N/S` с валидацией;
  - выбор файлов `.dxf/.dwg`;
  - при `.dwg` используется отдельный нативный парсер DWG;
  - при `.dxf/.dwg` файл копируется в `overlays/`, создаётся запись в `vector_overlays`, слой доступен в UI.

- [x] **7. Рендеринг на карте** — реализован в `src/components/map/MapCanvas.tsx`:
  - векторные слои рендерятся через Leaflet (`Polyline`/`CircleMarker`);
  - `opacity`/`visible`/`z_index` учитываются;
  - используется отдельный `Pane` с управлением `zIndex`.

- [x] **8. Управление слоем (LeftPanel)** — добавлен блок «Векторные слои» рядом с растровыми:
  - список `vector_overlays` с eye-toggle, opacity-slider, drag-to-reorder
  - аналогично существующему UI растровых слоёв в `src/components/map/LeftPanel.tsx`

- [x] **9. Тесты**:
  - `src/test/utm.test.ts` — unit: UTM ↔ WGS84 с контрольными точками (≥5 точек, разные зоны/полушария)
  - `src/test/parseDxf.test.ts` — unit: парсинг DXF-фрагментов, конвертация LINE/LWPOLYLINE/POLYLINE, отклонение бинарного контента
  - `src/test/topToolbar.test.tsx` — UI: сценарий `DWG/DXF + UTM` и передача параметров зоны/полушария
  - `src/test/missionRepositoryVectorOverlays.test.ts` — сериализация/десериализация `mission.ui.vector_overlays`

- [x] **10. Verify**:
  - `npm run typecheck`
  - `npm run lint`
  - `npm run test`

- [x] **11. Персистентный cache для векторных слоев**:
  - в `mission.ui.vector_overlays` добавлено поле `cache_file` с относительным путем `overlays/<id>.vector-cache.json`;
  - кэш хранит распарсенную WGS84-геометрию и bounds вместе с метаданными источника (`file/type/encoding/utm`);
  - при открытии миссии используется `cache-first`; при cache-miss/ошибке выполняется повторный нативный парсинг исходного CAD-файла и регенерация кэша;
  - при удалении слоя удаляются и `file`, и `cache_file`.
