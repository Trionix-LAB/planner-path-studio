# T-102: Импорт DXF/DWG на карту (координаты UTM из файла)

**Issue:** #102
**Spec:** New requirement (R-062)

## Context

* Карта уже поддерживает растровые наложения (GeoTIFF / TFW) через `raster_overlays` в `MissionUiState`.
* Векторных overlay-слоёв нет; DXF/DWG не поддерживаются.
* Внутренний CRS карты — WGS84 (EPSG:4326); вся геометрия хранится в этом CRS.
* DXF/DWG-файлы с инженерными чертежами задают координаты в UTM (метры, зона + полушарие).
  - В MVP: ручная геопривязка (GCP/аффинные матрицы) не требуется — достаточно конвертации UTM → WGS84.
* Уже есть `src/features/geo/crs.ts`; UTM <-> WGS84 конвертацию следует добавить туда же
  (либо отдельным модулем `src/features/geo/utm.ts`).
* DWG — проприетарный бинарный формат; без тяжёлых зависимостей полная поддержка невозможна.
  MVP ограничивается **DXF**; при попытке открыть DWG показывать подсказку «Экспортируйте в DXF».

## Goal

* Импорт DXF-файла как пользовательского векторного overlay-слоя поверх карты.
* Геометрия из DXF позиционируется на карте через конвертацию UTM → WGS84.
* UI управления слоем: visible / opacity / z-order (аналогично raster overlays).
* Слой сохраняется в папке миссии и восстанавливается при повторном открытии.
* Метаданные слоя хранятся в `mission.ui.vector_overlays`.

## Implementation plan

- [ ] **1. Spec** — добавить `R-062` в `spec/spec.md`:
  - внутренний CRS: WGS84 (EPSG:4326)
  - поддерживаемый входной CRS для DXF: UTM (зона 1–60, полушарие N/S)
  - правило выбора UTM-параметров: из файла (заголовок `$EXTMIN`/`$EXTMAX` или CRS-аннотация), иначе — диалог
  - единицы DXF: метры (MEASUREMENT=1); при несовпадении — ошибка или предупреждение с настройкой масштаба
  - модель хранения: файл копируется в `<mission_dir>/overlays/<id>.dxf`; метаданные — в `mission.ui.vector_overlays[]`

- [ ] **2. Зависимость: DXF-парсер** — добавить лёгкую JS-библиотеку для чтения DXF
  (например `dxf-parser` или `@dxfjs/dxf`); проверить что подходит для браузера/Electron без нативных зависимостей.

- [ ] **3. UTM → WGS84 конвертация** — реализовать `src/features/geo/utm.ts`:
  - `utmToWgs84({ easting, northing, zone, hemisphere }): { lat, lon }`
  - `wgs84ToUtm({ lat, lon }): { easting, northing, zone, hemisphere }`
  - pure, unit-testable, без side-effects

- [ ] **4. DXF reader** — реализовать `src/features/map/dxfOverlay/parseDxf.ts`:
  - принимает строку (содержимое `.dxf`)
  - возвращает `DxfFeatureCollection` (GeoJSON-подобная структура, геометрия в WGS84)
  - поддерживаемые примитивы MVP: `LINE`, `LWPOLYLINE`, `POLYLINE`, `POINT`, `INSERT` (block reference — базовый)
  - конвертирует каждую точку UTM → WGS84 (зона/полушарие передаются параметром)
  - если единицы DXF != метры: возвращает `ParseError` с кодом `UNITS_NOT_METERS`

- [ ] **5. Типы и хранение** — расширить `MissionUiState` в `src/features/mission/model/types.ts`:
  ```ts
  vector_overlays?: Array<{
    id: string;
    name: string;
    file: string;          // относительный путь в папке миссии
    type: 'dxf';
    utm_zone: number;      // 1..60
    utm_hemisphere: 'N' | 'S';
    opacity: number;       // 0..1
    visible: boolean;
    z_index: number;
  }>;
  ```
  - обновить `buildMissionBundle` / persistence path в `MapWorkspace.tsx`

- [ ] **6. Импорт UI** — добавить кнопку/пункт меню «Импорт DXF/DWG» (рядом с растровым импортом):
  - `src/components/map/TopToolbar.tsx` или меню миссии
  - `src/components/dialogs/ImportDxfDialog.tsx`:
    - выбор файла (`.dxf`, `.dwg`)
    - при `.dwg` — показать подсказку «Экспортируйте в DXF» и заблокировать импорт
    - при `.dxf` — попытаться определить UTM из файла; если не найдено — показать поля `zone` (1–60) и `hemisphere` (N/S)
    - предупреждение если единицы != метры (с возможностью указать коэффициент масштаба)
    - кнопка «Импортировать»
  - при успехе: копировать файл в `overlays/`, добавить запись в `vector_overlays`, вызвать `onVectorOverlayAdd`

- [ ] **7. Рендеринг на карте** — добавить `src/components/map/DxfOverlayLayer.tsx`:
  - принимает `DxfFeatureCollection` (WGS84) + `opacity` + `visible`
  - рендерит примитивы через Leaflet (`L.polyline`, `L.circle`) или через SVG overlay
  - z-order: `pane` Leaflet с кастомным `zIndex`

- [ ] **8. Управление слоем (LeftPanel)** — добавить блок «Векторные слои» рядом с растровыми:
  - список `vector_overlays` с eye-toggle, opacity-slider, drag-to-reorder
  - аналогично существующему UI растровых слоёв в `src/components/map/LeftPanel.tsx`

- [ ] **9. Тесты**:
  - `src/test/utm.test.ts` — unit: UTM ↔ WGS84 с контрольными точками (≥5 точек, разные зоны/полушария)
  - `src/test/parseDxf.test.ts` — unit: парсинг DXF-фрагментов, конвертация LINE/LWPOLYLINE, ошибка при неверных единицах
  - `src/test/importDxfDialog.test.tsx` — UI: выбор .dwg → подсказка; выбор .dxf без UTM в заголовке → показ полей зоны

- [ ] **10. Verify**:
  - `npm run typecheck`

  - `npm run test`
