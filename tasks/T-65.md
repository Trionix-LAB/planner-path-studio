# T-65: notifications not shown on top-most layer

**Issue:** #65  
**Spec:** New requirement (UI layering for system notifications)

## Context / Findings
- `ToastViewport` uses `z-[100]` in `src/components/ui/toast.tsx`.
- Dialog overlays/content use `z-[3000]`/`z-[3001]` in `src/components/ui/dialog.tsx`.
- Map transient overlays/context menus use up to `z-[9999]` in `src/components/map/MapCanvas.tsx`.
- Result: toast notifications can be rendered below active UI layers.

## Goal
- System notifications (save/error/info toasts) are always visible above application UI layers.

## Acceptance
- Notifications are displayed on the top-most layer and are not covered by dialogs, map overlays, or other UI elements.

## Plan
- [x] Add a regression test for notification layering (toast viewport class / stacking order contract).
- [x] Increase toast layer priority above all current app overlays (`dialog`, map overlays, context menus).
- [x] Verify there is no conflicting duplicate notification container in `App.tsx` and keep a single deterministic stack order.
- [ ] Validate behavior manually on map + dialog scenarios (open dialog + trigger toast).
- [x] Run checks: `npm run typecheck`, `npm run lint`, `npm run test`.
