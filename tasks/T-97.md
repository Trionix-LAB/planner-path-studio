# T-97: Import and manage GeoTIFF / TFW raster overlays

**Issue:** #97
**Spec:** New requirement

## Context
The map is Leaflet-based (`react-leaflet`): `src/components/map/MapCanvas.tsx`.
* Basemap is a `CachedTileLayer` (OSM) + optional overlay `TileLayer`.
* Layer visibility is controlled via `layers` state in `src/pages/MapWorkspace.tsx` and settings UI.
* Mission persistence uses `MissionUiState` (`src/features/mission/model/types.ts`) stored inside `mission.json` via repository bundle save/load.

There is currently no concept of user-imported raster overlays.

## Goal
Add support for importing georeferenced TIFF rasters and managing them as independent map overlays:
* Import:
  - GeoTIFF `.tif` (with embedded georeference)
  - `.tif + .tfw` (world file)
  - via file picker and drag-and-drop
* Render as separate overlay layers above/below basemap.
* Per overlay: visibility toggle, opacity slider, z-order controls.
* Persist overlay list + settings in mission and restore on open.
* Error UX for missing georef / unsupported CRS / corrupted images.

## MVP constraints (recommended)
* CRS support:
  - GeoTIFF: only `EPSG:4326` (WGS84).
  - `TIF + TFW`: import modes `градусы` (`EPSG:4326`), `Меркатор` (`EPSG:3857`) и `UTM зона` (1..60 + `N/S`) with conversion to `EPSG:4326` for rendering.
* `TFW` format contract:
  - world file must contain exactly 6 rows in order `A, D, B, E, C, F`;
  - `C/F` are coordinates of the center of top-left pixel;
  - for MVP rotation is not supported (`B/D != 0` -> explicit import error).
* Storage strategy:
  - Store raster bytes in mission directory (not inside mission.json).
  - Persist only metadata (file path, bounds, opacity, visible, z-index, name) in mission ui.

Rationale: avoids huge JSON, supports offline missions, and fits the existing FileStore model.

## Proposed data model
Add to `MissionUiState`:
* `raster_overlays?: Array<{
    id: string;
    name: string;
    file: string; // path under mission root, e.g. `overlays/<id>.tif`
    bounds: { north: number; south: number; east: number; west: number };
    opacity: number; // 0..1
    visible: boolean;
    z_index: number;
    source: 'geotiff' | 'tif+tfw';
  }>`

## Implementation plan (agent)
1) Spec + docs
* Add a new section in `spec/spec.md` (R-0xx) describing:
  - formats supported
  - CRS limitations for MVP
  - persistence strategy (bytes in mission dir + metadata in mission.json)
  - UI controls (visibility/opacity/order)
* Add a short user doc under `docs/` (e.g. `docs/features/raster-overlays.md`) with import steps and limitations.

2) GeoTIFF / TFW parsing
* Add dependency: `geotiff` (geotiff.js) for reading GeoTIFF in the renderer.
* Create `src/features/map/rasterOverlays/`:
  - `parseGeoTiff.ts`: read ArrayBuffer, extract:
    - image width/height
    - tiepoints/pixel scale OR geokeys to compute bounds
    - CRS/GeoKeyDirectory info to validate EPSG:4326
  - `parseTfw.ts`: parse 6-line worldfile and compute bounds for an image.
  - `computeBounds.ts`: helpers.
* Output of parsing: `{ bounds, width, height, crs }`.

3) Storage into mission directory
* In `MapWorkspace` import flow:
  - read selected file(s) as ArrayBuffer
  - write `.tif` bytes to mission root under `overlays/` using `platform.fileStore.writeBinary` (if not present, add binary support; otherwise base64 fallback).
  - store relative path `overlays/<id>.tif` in overlay meta.

4) Rendering on Leaflet
* Implement `RasterOverlayLayer` component:
  - use Leaflet `L.ImageOverlay` with a blob URL created from stored bytes.
  - Provide `opacity` and `zIndex`.
  - For large rasters: consider downsample to canvas and use PNG blob URL for display.

5) UI for overlay management
* Add a new section in LeftPanel layers or a dedicated overlays panel:
  - master toggle for basemap
  - list of imported overlays with:
    - visibility checkbox
    - opacity slider
    - reorder controls (drag handle or up/down)
    - delete overlay
  - import button + drag&drop zone

6) Mission persistence
* On save: include `raster_overlays` in `MissionUiState`.
* On open: load overlays metadata and render them.
  - Ensure missing files are handled (show warning and skip).

7) Error handling
* Explicit errors:
  - missing georeference
  - unsupported CRS
  - parse failure
  - file too large
* Show errors via toast/dialog with next steps.

8) Tests
* Unit tests:
  - `parseTfw` for known inputs
  - bounds computation
  - serialization/deserialization of `raster_overlays`
* Optional integration:
  - mock file import pipeline and ensure overlay meta is added and persisted.

9) Verify
* `npm run typecheck`
* `npm run lint`
* `npm run test`

## Acceptance checklist
* Импорт доступен из меню миссии двумя отдельными пунктами: `TIF (встроенная привязка)` и `TIF + TFW`.
* GeoTIFF и `.tif + .tfw` импортируются в соответствующих режимах.
* Для `TIF + TFW` доступны режимы импорта `градусы`, `Меркатор (EPSG:3857)` и `UTM зона`.
* Для `TIF + TFW` пользователь выбирает только `tif/tiff`; одноименный `tfw` автоматически ищется в той же папке для каждого выбранного `tif`.
* Для `TIF + TFW` парсер строго поддерживает world file из 6 строк (`A,D,B,E,C,F`) и корректно интерпретирует `C/F` как центр верхнего левого пикселя.
* Overlay renders in correct bounds.
* Per-overlay visibility, opacity, and z-order work.
* State persists in mission and restores.
* Clear errors for invalid files.
