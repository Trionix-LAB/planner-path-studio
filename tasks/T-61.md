# T-61: Fix "Regenerate legs" button

**Issue:** #61
**Spec:** R-048

## Why
* Currently, changing lane generation parameters in the sidebar doesn't update the lanes until they are saved and then regenerated. Users find it confusing when the "Regenerate" button doesn't seem to use the values they just entered.
* Redundant "Regenerate" button in the warning block clutter the UI.

## Goal
* The main "Regenerate lanes" button in the sidebar will automatically save current parameters before regenerating.
* The "Regenerate" button within the "Lanes are outdated" warning will be removed.
* Ensure regeneration works correctly with the latest values.

## Plan
* [ ] Step 1: Remove "Regenerate" button from the `zoneLanesOutdated` warning in `MapObjectProperties.tsx`.
* [ ] Step 2: Update `MapObjectProperties.tsx` to call `onSave` before `onRegenerateLanes` when the main button is clicked, or update the `onRegenerateLanes` interface to pass current parameters.
* [ ] Step 3: Ensure `handleRegenerateLanes` in `MapWorkspace.tsx` handles the latest object state correctly.
* [ ] Step 4: Verify that clicking "Regenerate lanes" after changing parameters (without manual saving) updates the lanes on the map.

## Acceptance
* [ ] Main "Regenerate lanes" button successfully updates lanes using current sidebar values (even if not manually saved).
* [ ] Warning "Галсы неактуальны" appears when parameters change, but has no button inside.
* [ ] No regressions in context menu regeneration.
