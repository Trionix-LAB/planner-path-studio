## Why

`GridLayer` рассчитывает шаг сетки через простое приближение:

```ts
// src/components/map/GridLayer.tsx, строки 17–20
function metersToDegrees(meters: number, lat: number): { dLat: number; dLon: number } {
  const dLat = meters / EARTH_RADIUS * RAD_TO_DEG;
  const dLon = meters / (EARTH_RADIUS * Math.cos(lat * DEG_TO_RAD)) * RAD_TO_DEG;
  return { dLat, dLon };
}
```

Затем сетка строится как равноотстоящие горизонтальные/вертикальные линии в lat/lon:

```ts
for (let lat = startLat; lat <= endLat; lat += dLat) {
  lines.push([[lat, west], [lat, east]]);
}
for (let lon = startLon; lon <= endLon; lon += dLon) {
  lines.push([[south, lon], [north, lon]]);
}
```

**Проблема:** `dLon = meters / (R · cos(lat))` — это сферическое приближение, корректное только у экватора. Оно:
- не учитывает сплюснутость эллипсоида WGS84;
- накапливает ошибку при больших latitude (особенно > 55°);
- не привязывает линии к реальной метрической сетке UTM.

Результат — линии сетки смещены относительно истинных метрических координат, ошибка может превышать допуск 1% (R-002).

---

## Problem analysis

### Где проблема

**`src/components/map/GridLayer.tsx` — вся логика вычисления линий:**

1. `metersToDegrees` (строки 17–20) — сферическое приближение вместо UTM-проекции.
2. `snapToGrid` (строки 23–25) — привязка к сетке работает в градусах (`snapToGrid(south, dLat)`), что не соответствует метрической сетке UTM.
3. Горизонтальные линии (`lat += dLat`) — линии параллелей, они не совпадают с горизонтальными линиями UTM-сетки (которые идут параллельно оси northing).
4. Вертикальные линии (`lon += dLon`) — аналогично, не совпадают с вертикальными линиями UTM-сетки (easting).

### Что уже есть в проекте

`src/features/geo/utm.ts` — полноценная реализация UTM-проекции (без внешних зависимостей):
- `wgs84ToUtm(point)` — конвертация WGS84 → UTM easting/northing/zone/hemisphere
- `utmToWgs84(point)` — обратная конвертация UTM → WGS84
- Ограничение: работает для широт -80..84°

Тесты: `src/test/utm.test.ts` — проверяет round-trip точность.

## Goal

Переработать логику построения сетки в `GridLayer.tsx` так, чтобы:
1. Линии сетки соответствовали метрической сетке UTM текущей зоны.
2. Привязка (snap) к сетке выполнялась в пространстве easting/northing.
3. Погрешность визуальной сетки ≤ 1% на экране (R-002).
4. Авто и ручной шаг сетки (`step` prop) сохраняли работоспособность.

## Solution strategy

### Алгоритм построения UTM-сетки

Для каждого обновления карты (`moveend`, `zoomend`):

1. **Определить UTM-зону** по текущему центру карты:
   ```ts
   const center = map.getCenter();
   const utmCenter = wgs84ToUtm({ lat: center.lat, lon: center.lng });
   // utmCenter.zone, utmCenter.hemisphere
   ```

2. **Конвертировать углы вьюпорта в UTM** (все четыре угла bounds → UTM, взять min/max easting/northing):
   ```ts
   const bounds = map.getBounds();
   const corners = [
     { lat: bounds.getNorth(), lon: bounds.getWest() },
     { lat: bounds.getNorth(), lon: bounds.getEast() },
     { lat: bounds.getSouth(), lon: bounds.getWest() },
     { lat: bounds.getSouth(), lon: bounds.getEast() },
   ];
   // Конвертировать каждый угол в UTM той же зоны (zone, hemisphere центра)
   // minEasting = Math.min(...corners.map(c => wgs84ToUtmForced(c, zone, hemisphere).easting))
   // maxEasting = ...
   // minNorthing = ...
   // maxNorthing = ...
   ```

   **Важно:** все углы должны проецироваться в **одну и ту же** UTM-зону (зону центра), иначе линии будут рваться на стыках зон.

3. **Snap к кратному step в UTM**:
   ```ts
   const startEasting = Math.floor(minEasting / step) * step;
   const startNorthing = Math.floor(minNorthing / step) * step;
   ```

4. **Построить вертикальные линии (easting = const)**: для каждого `e = startEasting; e <= maxEasting; e += step` построить пару точек `(minNorthing, e)` и `(maxNorthing, e)`, конвертировать в WGS84.

5. **Построить горизонтальные линии (northing = const)**: для каждого `n = startNorthing; n <= maxNorthing; n += step` построить пару точек `(n, minEasting)` и `(n, maxEasting)`, конвертировать в WGS84.

6. **Конвертировать обратно в WGS84** через `utmToWgs84`.

### Граничный случай: широты > 84° и < -80°

UTM не определён для полярных широт (> 84° N и < 80° S). Там `wgs84ToUtm` бросает ошибку. В таких случаях нужно безопасно вернуть пустой массив линий.

Добавить guard:
```ts
if (Math.abs(center.lat) > 80) {
  setGridLines([]);
  return;
}
```

### Структура новых функций

Вынести всю UTM-логику из компонента в отдельный модуль `src/components/map/gridUtils.ts`:

```ts
import { wgs84ToUtm, utmToWgs84, type UtmHemisphere } from '@/features/geo/utm';

export type GridLine = [[number, number], [number, number]]; // [[lat, lon], [lat, lon]]

export type GridBoundsUtm = {
  zone: number;
  hemisphere: UtmHemisphere;
  minEasting: number;
  maxEasting: number;
  minNorthing: number;
  maxNorthing: number;
};

/**
 * Проецирует WGS84 bounds в UTM-координаты относительно заданной зоны.
 * Все углы bbox проецируются в одну зону (зону центра).
 * Возвращает null если центр карты вне диапазона UTM (-80..84°).
 */
export function boundsToUtm(
  centerLat: number,
  centerLon: number,
  corners: Array<{ lat: number; lon: number }>,
): GridBoundsUtm | null { ... }

/**
 * Строит линии метрической сетки в WGS84 через UTM-проекцию.
 * step — шаг в метрах.
 * padding — количество дополнительных шагов за границами вьюпорта (default: 1).
 */
export function buildUtmGridLines(
  utmBounds: GridBoundsUtm,
  step: number,
  padding?: number,
): GridLine[] { ... }
```

**`boundsToUtm` — детали:**
- Вычислить UTM-зону по центру: `wgs84ToUtm({ lat: centerLat, lon: centerLon })`.
- Для каждого угла: вызвать `wgs84ToUtm` и принудительно перепривязать к зоне центра. Стандартный `wgs84ToUtm` возвращает свою зону — нам нужно перепроецировать в зону центра. Для этого: взять easting/northing из `wgs84ToUtm` угла, но заменить zone/hemisphere на zone/hemisphere центра. **Это корректно** пока вьюпорт не пересекает границу зоны более чем на ~200–300 км — для карты на экране это всегда выполняется.
- Вернуть `{ zone, hemisphere, minEasting, maxEasting, minNorthing, maxNorthing }`.

**`buildUtmGridLines` — детали:**
- `padding` линий за пределы вьюпорта (по умолчанию 1) — чтобы линии доходили до края экрана при пан/зум.
- Вертикальные линии: easting = const, от `(minNorthing - padding*step)` до `(maxNorthing + padding*step)`.
- Горизонтальные линии: northing = const, от `(minEasting - padding*step)` до `(maxEasting + padding*step)`.
- Каждая точка конвертируется через `utmToWgs84`.
- При любой ошибке конвертации — пропустить линию (не добавлять в результат).

```ts
const updateGrid = () => {
  const bounds = map.getBounds();
  const center = bounds.getCenter();
  const currentStep = step ?? computeScaleFromMap(map).distanceM;

  const corners = [
    { lat: bounds.getNorth(), lon: bounds.getWest() },
    { lat: bounds.getNorth(), lon: bounds.getEast() },
    { lat: bounds.getSouth(), lon: bounds.getWest() },
    { lat: bounds.getSouth(), lon: bounds.getEast() },
  ];

  const utmBounds = boundsToUtm(center.lat, center.lng, corners);
  if (!utmBounds) {
    setGridLines([]);
    return;
  }

  setGridLines(buildUtmGridLines(utmBounds, currentStep));
};
```

### Шаг 3: Тесты `src/test/gridUtils.test.ts` (НОВЫЙ файл)

```ts
import { describe, expect, it } from 'vitest';
import { boundsToUtm, buildUtmGridLines } from '@/components/map/gridUtils';
import { haversineDistanceMeters } from '@/components/map/scaleUtils';

describe('boundsToUtm', () => {
  it('возвращает корректную зону для центра карты (Санкт-Петербург, зона 36N)', () => {
    const result = boundsToUtm(59.9, 30.3, [
      { lat: 60.0, lon: 30.0 }, { lat: 60.0, lon: 30.6 },
      { lat: 59.8, lon: 30.0 }, { lat: 59.8, lon: 30.6 },
    ]);
    expect(result).not.toBeNull();
    expect(result!.zone).toBe(36);
    expect(result!.hemisphere).toBe('north');
    expect(result!.minEasting).toBeLessThan(result!.maxEasting);
    expect(result!.minNorthing).toBeLessThan(result!.maxNorthing);
  });

  it('возвращает null для широт > 80°', () => {
    const result = boundsToUtm(85, 0, [
      { lat: 85, lon: -1 }, { lat: 85, lon: 1 },
      { lat: 84, lon: -1 }, { lat: 84, lon: 1 },
    ]);
    expect(result).toBeNull();
  });

  it('возвращает null для широт < -80°', () => {
    const result = boundsToUtm(-81, 0, [
      { lat: -80, lon: -1 }, { lat: -80, lon: 1 },
      { lat: -82, lon: -1 }, { lat: -82, lon: 1 },
    ]);
    expect(result).toBeNull();
  });
});

describe('buildUtmGridLines', () => {
  it('возвращает пустой массив при step <= 0', () => {
    const bounds = boundsToUtm(59.9, 30.3, [
      { lat: 60.0, lon: 30.0 }, { lat: 60.0, lon: 30.6 },
      { lat: 59.8, lon: 30.0 }, { lat: 59.8, lon: 30.6 },
    ])!;
    expect(buildUtmGridLines(bounds, 0)).toEqual([]);
    expect(buildUtmGridLines(bounds, -100)).toEqual([]);
  });

  it('ошибка шага ≤ 1% для step=100м на широте ~60° (Санкт-Петербург)', () => {
    // Построить сетку, взять две соседние вертикальные линии,
    // измерить расстояние между ними через haversineDistanceMeters,
    // проверить что |measured - step| / step <= 0.01
    const step = 100;
    const bounds = boundsToUtm(59.9, 30.3, [
      { lat: 60.1, lon: 29.9 }, { lat: 60.1, lon: 30.7 },
      { lat: 59.7, lon: 29.9 }, { lat: 59.7, lon: 30.7 },
    ])!;
    const lines = buildUtmGridLines(bounds, step);
    // Фильтровать вертикальные линии (lat[0] == lat[1] — нет, наоборот lon[0] == lon[1] для вертикальных)
    // Проверить расстояние между двумя соседними точками одной линии (northing direction)
    // Или взять горизонтальные линии и измерить расстояние между ними
    expect(lines.length).toBeGreaterThan(0);
    // Взять первые две горизонтальные линии (northing = const) и измерить разницу по широте
    const hLines = lines.filter(([[lat1, lon1], [lat2, lon2]]) => Math.abs(lon1 - lon2) > 0.001);
    if (hLines.length >= 2) {
      const lat1 = hLines[0][0][0];
      const lat2 = hLines[1][0][0];
      const distM = haversineDistanceMeters(lat1, 30.3, lat2, 30.3);
      const error = Math.abs(distM - step) / step;
      expect(error).toBeLessThanOrEqual(0.01);
    }
  });

  it('ошибка шага ≤ 1% для step=50м на широте ~0° (экватор)', () => {
    const step = 50;
    const bounds = boundsToUtm(0.1, 0.1, [
      { lat: 0.2, lon: 0.0 }, { lat: 0.2, lon: 0.2 },
      { lat: 0.0, lon: 0.0 }, { lat: 0.0, lon: 0.2 },
    ])!;
    const lines = buildUtmGridLines(bounds, step);
    expect(lines.length).toBeGreaterThan(0);
    const hLines = lines.filter(([[lat1, lon1], [, lon2]]) => Math.abs(lon1 - lon2) > 0.0001);
    if (hLines.length >= 2) {
      const lat1 = hLines[0][0][0];
      const lat2 = hLines[1][0][0];
      const distM = haversineDistanceMeters(lat1, 0.1, lat2, 0.1);
      const error = Math.abs(distM - step) / step;
      expect(error).toBeLessThanOrEqual(0.01);
    }
  });

  it('ошибка шага ≤ 1% для step=200м на широте ~-33° (Буэнос-Айрес)', () => {
    const step = 200;
    const bounds = boundsToUtm(-34.6, -58.4, [
      { lat: -34.5, lon: -58.5 }, { lat: -34.5, lon: -58.3 },
      { lat: -34.7, lon: -58.5 }, { lat: -34.7, lon: -58.3 },
    ])!;
    const lines = buildUtmGridLines(bounds, step);
    expect(lines.length).toBeGreaterThan(0);
  });

  it('линии имеют ровно 2 точки каждая', () => {
    const bounds = boundsToUtm(59.9, 30.3, [
      { lat: 60.0, lon: 30.0 }, { lat: 60.0, lon: 30.6 },
      { lat: 59.8, lon: 30.0 }, { lat: 59.8, lon: 30.6 },
    ])!;
    const lines = buildUtmGridLines(bounds, 100);
    for (const line of lines) {
      expect(line).toHaveLength(2);
      expect(line[0]).toHaveLength(2);
      expect(line[1]).toHaveLength(2);
    }
  });

  it('количество линий разумно (не слишком много, не слишком мало) для zoom~15', () => {

      { lat: 59.905, lon: 30.29 }, { lat: 59.905, lon: 30.31 },
      { lat: 59.895, lon: 30.29 }, { lat: 59.895, lon: 30.31 },
    ])!;

    expect(lines.length).toBeGreaterThan(4);
    expect(lines.length).toBeLessThan(100);
  });
});
```

## Guardrails

- **Все углы вьюпорта проецировать в одну UTM-зону** (зону центра карты) — нельзя смешивать easting/northing из разных зон.
- **UTM не работает для |lat| > 80°** — `wgs84ToUtm` бросает ошибку; guard: если `Math.abs(centerLat) > 80`, возвращать пустой массив линий без ошибки в консоль.
- **Не менять `GridLayerProps`** — интерфейс компонента (`visible`, `step`, `color`, `widthPx`, `lineStyle`) остаётся прежним.
- **`step` prop и авто-шаг** (`computeScaleFromMap(map).distanceM`) — оба должны работать. Изменяется только способ перевода метрового шага в линии на карте, не источник шага.
- **Не добавлять внешние зависимости** — только `src/features/geo/utm.ts` (уже в проекте).
- **Не менять JSX рендеринг** `<Polyline>` — меняется только массив `gridLines`, который в него передаётся.
- **Padding**: добавить 1 шаг за границами вьюпорта с каждой стороны — чтобы линии были видны при начале пана до следующего `moveend`.
