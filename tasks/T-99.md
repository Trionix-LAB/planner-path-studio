# T-99: Base station track (render + record controls + export)

**Issue:** #99
**Spec:** New requirement

## Context (current implementation)
The app already has a generalized per-agent track recording pipeline:

* Track storage model (mission): `src/features/mission/model/types.ts`
  - `mission.active_tracks: Record<string, string>` maps `agentId -> trackId`
  - `mission.tracks: MissionTrackMeta[]` stores track metadata (`agent_id`, `file`, timestamps)
  - `trackPointsByTrackId: Record<string, TrackPoint[]>` stores points and `segment_id`

* Recorder state machine: `src/features/mission/model/trackRecorder.ts`
  - events: `start | pause | resume | stop | fixReceived | connectionRestored`
  - **Important detail:** `pause`/`stop` closes the active track (removes entry from `active_tracks` and sets `ended_at`).
    A subsequent `start` creates a new track file (this is how "pause segmentation" works today for agents).

* Map integration:
  - `MapWorkspace` builds `trackSegments` from `trackPointsByTrackId` (`buildTrackSegments`) and assigns colors by `track.agent_id` -> diver.track_color.
    See: `src/pages/MapWorkspace.tsx` around `trackSegments` memo.

* Base station already exists as a marker + navigation source assignment:
  - `mission.ui.base_station` stores `navigation_source` and the latest telemetry snapshot.
  - `MapWorkspace` keeps `baseStationNavigationSource` and `baseStationTelemetry`.
  - Map marker rendering is in `MapCanvas` (`baseStationData` props).
  - Settings UI includes base station source selection in `SettingsDialog`.

Gap: base station fixes are not recorded into tracks, base station has no track controls/color, and export naming does not distinguish base station.

## Goal
* Add a base station track that behaves like agent tracks:
  - recorded from base station telemetry fixes
  - pause/finish controls in the left panel (where agents live)
  - track is rendered on the map with a configurable color
  - track is included in export and labeled meaningfully

## Key decisions (recommended)
1) Represent base station as a dedicated recorder "agent id":
   - constant `BASE_STATION_AGENT_ID = 'base-station'`
   - store base station tracks in `mission.tracks` with `agent_id = BASE_STATION_AGENT_ID`
   - store active track mapping in `mission.active_tracks[BASE_STATION_AGENT_ID]`

2) Store base station track color in mission UI state:
   - add `mission.ui.base_station.track_color: string` (hex)
   - default to `styles.track.color` or a dedicated default

This keeps the existing recorder/export pipeline intact and minimizes new storage concepts.

## Implementation plan (agent)
1) Spec

  - base station track exists and is managed similarly to agent tracks
  - storage: `agent_id = 'base-station'` and `mission.ui.base_station.track_color`
  - UI controls location (LeftPanel)
  - export includes base station

2) Mission UI state + normalization
* Extend `MissionUiState.base_station` type in `src/features/mission/model/types.ts` to include `track_color?: string`.
* Update bundle persistence in `MapWorkspace` (the `buildMissionBundle` call site already persists `base_station` snapshot) to include `track_color`.
* Ensure defaults are set when creating a new mission (`src/features/mission/model/repository.ts` default UI state).
* Update any normalization helpers if present for base_station fields.

3) Recording base station fixes
* In `src/pages/MapWorkspace.tsx`:
  - track base station recording status from `trackStatusByAgentId[BASE_STATION_AGENT_ID]`.
  - when `baseStationTelemetry` updates (or inside `syncBaseStationTelemetry` after set), dispatch:
    - `trackRecorderReduce(prev, { type: 'fixReceived', agentId: BASE_STATION_AGENT_ID, fix: ... })`
    - only when status is `recording`.
  - de-dup by `received_at` similarly to divers (add `lastRecordedFixByAgentRef.current[BASE_STATION_AGENT_ID]`).

4) Left panel controls
* Update `src/components/map/LeftPanel.tsx` and `src/pages/MapWorkspace.tsx` wiring:
  - Add a base station row in the Agents section (or a small "Базовая станция" section above agents) showing:
    - status indicator (recording/paused/stopped)
    - buttons:
      - "Пауза" when recording (dispatch `pause`)
      - "Продолжить" when paused/stopped (dispatch `start`)
      - "Завершить" (dispatch `stop`)
  - Keep controls disabled when recording is not enabled, similar to agent controls.
  - Controls MUST use the same style-system as agent recording controls (same visual tokens, button sizing, hover/disabled behavior).
  - Click on the base station row in LeftPanel should select it as the active track owner for the RightPanel tracks list.

5) Track color UI (SettingsDialog)
* In `src/components/dialogs/SettingsDialog.tsx`:
  - Add a color input for base station track color under the "Базовая станция" block (same dialog users use for navigation sources).
  - Persist the value into mission ui state (via MapWorkspace settings save handler).

6) Track rendering color mapping
* In `MapWorkspace` `trackSegments` memo:
  - extend `agentColorByUid` to include `BASE_STATION_AGENT_ID -> baseStationTrackColor`.
  - baseStationTrackColor comes from `mission.ui.base_station.track_color` with a safe default.

7) Export naming
* In `handleExport` in `src/pages/MapWorkspace.tsx`:
  - Replace the generic `Трек N` naming:
    - if track.meta.agent_id matches a diver uid -> use diver title (e.g. `Маяк 1`)
    - if agent_id === BASE_STATION_AGENT_ID -> name as `Базовая станция` (optionally with index)
    - fallback to `Трек N`

10) Top toolbar + right panel parity
* Top toolbar track action (`start/pause/resume`) must affect base station recording as part of the common recorder action.
* Right panel tracks block must render base station tracks when base station is selected in left panel (same filtering/highlight behavior as agents).

11) Per-track color semantics
* Track color must be snapshotted per track in mission metadata.
* Changing agent/base-station color must affect only active (current) track and future tracks; completed tracks must keep historical color.

8) Tests
* Add unit/UI tests:
  - `trackRecorder` flow for base station id (start -> fixReceived -> pause/stop).
  - `MapWorkspace`/LeftPanel smoke: clicking base station pause/finish calls the correct reducer events.
  - Export: ensure base station track is included and named (can be unit-level if export builder is factored).
