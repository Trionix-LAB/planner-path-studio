# T-117: Hide empty Rasters and Vector layers sections in left panel

**Issue:** [#117](https://github.com/Trionix-LAB/planner-path-studio/issues/117)
**Branch:** `feat/ui-settings-117-hide-empty-overlay-sections`
**Spec:** `R-070` (добавить в `spec/spec.md`)

---

## Why

Секции «Растры» и «Векторные слои» всегда занимают место в левой панели, даже когда не содержат ни одного слоя. Для рабочих сессий без оверлеев это визуальный шум. Нужна опция, при включении которой пустые секции полностью исчезают из панели.

---

## Goal

Добавить флаг `hide_empty_overlay_sections` в глобальные настройки приложения. Когда включён — `LeftPanel` не рендерит секцию «Растры» и/или «Векторные слои» (включая заголовок и разделитель), если соответствующий массив оверлеев пуст. По умолчанию выключен — поведение идентично текущему.

---

## Scope (минимальный, spec-driven)

Задача маленькая: **3 файла изменений + 2 файла тестов**. Не добавлять ничего сверх описанного.

## Guardrails

- Не трогать `MissionUiState.left_panel_sections` — это collapse-состояние (свёрнут/развёрнут), не visibility.
- Не добавлять два раздельных флага (один на растры, один на векторы) — один флаг `hide_empty_overlay_sections` покрывает оба случая.
- Не менять `schema_version` — нормализатор безопасно работает с отсутствующим полем.
- Не изменять логику внутри секций — только условие рендеринга всего блока снаружи.
- Не создавать новые компоненты — изменения только в существующих файлах.
