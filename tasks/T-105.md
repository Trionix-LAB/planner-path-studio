# T-105: RWLT navigation system support (uNav RF Dongle)

**Issue:** #105
**Branch:** `feat/105-rwlt-unav-com`
**Spec:** New requirement — добавить R-XXX в `spec/spec.md` раздел «Оборудование»

---

## Why

Сейчас приложение поддерживает Zima2R, GNSS UDP и GNSS COM как источники
навигации. Система RWLT (uNav RWLT RF Dongle) — гидроакустический трекер
реального времени для подводных объектов (водолазы, ТНПА). Донгл
подключается по COM-порту и выдаёт NMEA0183 + проприетарные сообщения
UNV/RWL. Добавление поддержки позволит отображать позиции водолазов
(режим RedPhone-DX) и пингера в реальном времени без сторонних программ.

## Протокол uNav (источник истины)

**Физический уровень:**
```
Baudrate: 38400, Data bits: 8, Stop bits: 1, Parity: No, Flow control: No
```

**Контрольная сумма:** XOR всех байт между `$` и `*`, hex 2 символа.
Формула та же, что в `validateChecksum` в `gnss-udp/protocol.ts` — можно переиспользовать.

**Исходящие команды (renderer → донгл):**

| Сообщение | Формат | Когда отправлять |
|---|---|---|
| `$PUNV0,...*hh` | `$PUNV0,sty_PSU,wtmp_C,sos_mps,max_tspd_mps,sf_FIFO,sf_rthld,dhf_FIFO,dhf_rthld,ce_FIFO,brate,rwlt_mode,rwlt_drating*hh` | При подключении: установить `rwlt_mode` (0=pinger / 1=divers) согласно настройке пользователя. Остальные поля — пустые строки (донгл применит свои дефолты). |
| `$PUNV1,ref_type,ref_lat,ref_lon*hh` | ref_type=0 (AUX GNSS) | Опционально: задаёт опорную точку для расчёта дальности/курса. В MVP не отправлять. |

**Входящие сообщения от донгла — полная таблица:**

| Сообщение | Направление | Режим | Ключевые поля | Применение в приложении |
|---|---|---|---|---|
| `$GNGGA,...*hh` | донгл → app | pinger | [2..5]=lat/lon (ddmm), [6]=fixType, [8]=HDOP (радиальная ошибка, м), [9]=Altitude (глубина со знаком «-») | **Реализовать:** позиция + глубина пингера |
| `$GNRMC,...*hh` | донгл → app | pinger | [3..6]=lat/lon (ddmm), [8]=course | **Реализовать:** курс пингера |
| `$PUWV3,tID,tLat,tLon,tDpt,tCrs,tRer,Age*hh` | донгл → app | divers | tID=адрес водолаза (0..254), tLat/tLon=decimal degrees, tDpt=глубина (м), tCrs=курс (°), tRer=радиальная ошибка (м), Age=возраст (с) | **Реализовать:** позиция водолаза по фиксированной таблице beacon_id |
| `$PRWLA,bID,bLt,bLn,bDpt,bBat,pData,bTOA,bMSR*hh` | донгл → app | both | bID=номер буя (1-4), bLt/bLn=decimal degrees, bDpt=глубина антенны (м), bBat=напряжение (В), bTOA=время прихода сигнала (с), bMSR=SNR (дБ) | **Реализовать:** позиции 4 буёв → отображать на карте как вспомогательные маркеры (см. контракт ниже) |
| `$PUWV4,tID,rpLt,rpLn,dst2rp,crs2rp,crs4rp,Age*hh` | донгл → app | both | dst2rp=дальность до опорной точки (м), crs2rp/crs4rp=курсы | Парсить тип, **не использовать в MVP** (пропустить в провайдере) |
| `$PUWV5,gnssLt,gnssLn,gnssCrs,gnssSog*hh` | донгл → app | both | Координаты встроенного GNSS-приёмника донгла, курс, скорость (км/ч) | Парсить тип, **не использовать в MVP** |
| `$PUWV6,dataID,dataValue*hh` | донгл → app | both | Доп. данные от пингера (dataID определяет тип) | Парсить тип, **не использовать в MVP** |
| `$PUNV0,...*hh` | донгл → app | both | Эхо текущих настроек (ответ на запрос или при старте) | Парсить тип, **не использовать в MVP** |

**Глубина из GGA:** поле Altitude (индекс 9) передаётся со знаком «-» (отрицательное = под водой).
Нормализация: `depth = altitude < 0 ? -altitude : 0`.

**Режимы работы (`rwlt_mode`):**
- `0` (pinger): позиция обновляется из GGA/RMC каждые ~2 секунды. Один агент = пингер.
- `1` (divers): позиция каждого водолаза из PUWV3.tID → agentId. Обновление — после каждой PTT-передачи (нерегулярно).

  **Маппинг tID → agentId** (режим divers): **фиксированная таблица** — использовать `missionDivers` из состояния миссии.
  Найти `DiverUiConfig` где `beacon_id === String(tID)`, взять его `uid` как `entity_id`.
  Если совпадения нет → пропустить пакет (не публиковать fix для неизвестного tID).

### Шаг 2: Парсер протокола
- [ ] Создать `src/features/devices/rwlt-com/protocol.ts`.

  **Типы:**
  ```ts
  export type RwltMessageKind =
    | 'GGA'    // $GNGGA — позиция + глубина пингера
    | 'RMC'    // $GNRMC — позиция + курс пингера
    | 'PUWV3'  // позиция водолаза (режим divers)
    | 'PRWLA'  // данные от буя RWLT (оба режима)
    | 'PUWV4'  // параметры до опорной точки (не используется в MVP)
    | 'PUWV5'  // GNSS встроенного приёмника (не используется в MVP)
    | 'PUWV6'  // доп. данные пингера (не используется в MVP)
    | 'PUNV0'  // настройки донгла (не используется в MVP)
    | 'UNKNOWN';

  export type RwltGgaMessage = {
    kind: 'GGA';
    raw: string;
    lat: number;
    lon: number;
    depthM: number;          // нормализовано: altitude < 0 ? -altitude : 0
    radialErrorM: number | null; // поле HDOP (радиальная ошибка)
    hasFix: boolean;
  };

  export type RwltRmcMessage = {
    kind: 'RMC';
    raw: string;
    lat: number;
    lon: number;
    courseDeg: number | null;
    hasFix: boolean;
  };

  export type RwltPuwv3Message = {
    kind: 'PUWV3';
    raw: string;
    targetId: number;        // tID — адрес водолаза
    lat: number;             // decimal degrees
    lon: number;             // decimal degrees
    depthM: number;
    courseDeg: number | null;
    radialErrorM: number | null;
    ageSeconds: number | null;
  };

  // Данные от буя RWLT — контракт для отображения на карте
  export type RwltPrwlaMessage = {
    kind: 'PRWLA';
    raw: string;
    buoyId: number;          // bID, 1..4
    lat: number;             // decimal degrees
    lon: number;             // decimal degrees
    antennaDepthM: number;   // bDpt
    batteryV: number | null; // bBat
    toaS: number | null;     // bTOA — время прихода сигнала
    msrDb: number | null;    // bMSR — SNR в дБ
  };

  // Следующие типы парсируются, но провайдер их не обрабатывает в MVP
  export type RwltPuwv4Message = { kind: 'PUWV4'; raw: string };
  export type RwltPuwv5Message = { kind: 'PUWV5'; raw: string };
  export type RwltPuwv6Message = { kind: 'PUWV6'; raw: string };
  export type RwltPunv0Message = { kind: 'PUNV0'; raw: string };

  export type UnknownRwltMessage = { kind: 'UNKNOWN'; raw: string };

  export type ParsedRwltMessage =
    | RwltGgaMessage
    | RwltRmcMessage
    | RwltPuwv3Message
    | RwltPrwlaMessage
    | RwltPuwv4Message
    | RwltPuwv5Message
    | RwltPuwv6Message
    | RwltPunv0Message
    | UnknownRwltMessage;
  ```

  **Функции:**
  - `validateNmeaChecksum(line: string): string | null` — XOR checksum, вернуть payload или null.
    Можно скопировать из `gnss-udp/protocol.ts` (`validateChecksum`) или переиспользовать.
  - `parseCoordinate(coord: string, hemi: string, isLat: boolean): number | null` — ddmm.mmmm → decimal degrees.
    Идентична функции в `gnss-udp/protocol.ts` — скопировать/переиспользовать.
  - `parseRwltLine(line: string): ParsedRwltMessage` — главная точка входа.
  - `splitRwltDatagram(datagram: string): string[]` — split по `\r?\n`, trim, filter empty.

  **Логика parseRwltLine:**
  1. Валидировать checksum. Если невалидна → `UNKNOWN`.
  2. Разбить payload на поля по `,`.
  3. Определить тип по первому полю (sentenceId, без `$`):

  ```
  sentenceId оканчивается на 'GGA'  → parse GGA
  sentenceId оканчивается на 'RMC'  → parse RMC
  sentenceId === 'PUNV3'            → parse PUWV3   // $PUWV3 → payload без $ → PUNV3,...
  sentenceId === 'PRWLA'            → parse PRWLA   // $PRWLA → payload без $ → PRWLA,...
  sentenceId === 'PUNV4'            → { kind: 'PUWV4', raw }
  sentenceId === 'PUNV5'            → { kind: 'PUWV5', raw }
  sentenceId === 'PUNV6'            → { kind: 'PUWV6', raw }
  sentenceId === 'PUNV0'            → { kind: 'PUNV0', raw }
  иначе                             → { kind: 'UNKNOWN', raw }
  ```

  **parse GGA:**
  - lat = parseCoordinate(fields[2], fields[3], true)
  - lon = parseCoordinate(fields[4], fields[5], false)
  - fixQuality = parseNumber(fields[6]); hasFix = fixQuality > 0 && lat != null && lon != null && !(lat===0 && lon===0)
  - altitude = parseNumber(fields[9]); depthM = altitude !== null && altitude < 0 ? -altitude : 0
  - radialErrorM = parseNumber(fields[8])

  **parse RMC:**
  - status = fields[2]; hasFix = status === 'A' && lat != null && lon != null && !(lat===0 && lon===0)
  - lat = parseCoordinate(fields[3], fields[4], true)
  - lon = parseCoordinate(fields[5], fields[6], false)
  - courseDeg = parseNumber(fields[8])

  **parse PUWV3** (`$PUWV3,tID,tLat,tLon,tDpt,tCrs,tRer,Age`):
  - targetId = parseInt(fields[1]) — целое, 0..254
  - lat = parseNumber(fields[2]) — **decimal degrees, не ddmm!**
  - lon = parseNumber(fields[3]) — **decimal degrees, не ddmm!**
  - depthM = parseNumber(fields[4]) ?? 0
  - courseDeg = parseNumber(fields[5])
  - radialErrorM = parseNumber(fields[6])
  - ageSeconds = parseNumber(fields[7])
  - Если targetId невалиден или lat/lon = null → `UNKNOWN`

  **parse PRWLA** (`$PRWLA,bID,bLt,bLn,bDpt,bBat,pData,bTOA,bMSR`):
  - buoyId = parseInt(fields[1]) — 1..4
  - lat = parseNumber(fields[2]) — **decimal degrees**
  - lon = parseNumber(fields[3]) — **decimal degrees**
  - antennaDepthM = parseNumber(fields[4]) ?? 0
  - batteryV = parseNumber(fields[5])
  - (fields[6] = pData — пропустить)
  - toaS = parseNumber(fields[7])
  - msrDb = parseNumber(fields[8])
  - Если buoyId < 1 || buoyId > 4 или lat/lon = null → `UNKNOWN`

  > **Важно:** Координаты в `PUWV3` и `PRWLA` передаются как **decimal degrees**
  > (не ddmm.mmmm как в GGA/RMC). Не применять `parseCoordinate` — использовать `parseNumber`.

    intervalMs?: number;    // default 2000 (пингер обновляется ~0.5 Гц)
    timeoutMs?: number;     // default 5000
    mode: 'pinger' | 'divers';
    // Для режима divers — список agentId водолазов для симуляции (берётся из missionDivers).
    // Провайдер симулирует позицию для каждого из них.
    diverAgentIds?: Array<{ uid: string; id: string }>;
    // Начальная позиция (центр симуляции)
    startLat?: number;      // default 59.93428
    startLon?: number;      // default 30.335099
  };
  ```

  **Поведение — режим pinger:**
  - Каждые `intervalMs` публиковать один `TelemetryFix`:
    - Движение: та же синусоидальная формула что в `createSimulationTelemetryProvider`
      (`lat += 0.00003 × sin(tick/6)`, `lon += 0.00003 × cos(tick/6)`).
    - `depth = max(0, 12 + 2 × sin(tick/5))` — глубина осциллирует около 12 м.
    - `course = (course + 12) % 360`.
    - `entity_type: 'base_station'`, `entity_id: 'base-station'`, `source: 'SIM'`,
      `navigation_source_id: 'rwlt-com'`.
  - Имитировать пакетные потери: каждые 35 тиков — 7 тиков тишины (как в обычном симуляторе,
    строки 278–288 `telemetry.ts`).

  **Поведение — режим divers:**
  - Каждые `intervalMs` публиковать по одному `TelemetryFix` для каждого агента из `diverAgentIds`.
  - У каждого водолаза независимая фаза движения: `phaseOffset = index × (Math.PI / 3)`.
    ```
    lat = startLat + 0.0002 × sin(tick/8 + phaseOffset)
    lon = startLon + 0.0002 × cos(tick/8 + phaseOffset)
    depth = max(0, 10 + 3 × sin(tick/5 + phaseOffset))
    course = (45 + index × 60 + tick × 8) % 360
    ```
  - `entity_type: 'diver'`, `entity_id: diver.uid`, `source: 'SIM'`,
    `navigation_source_id: 'rwlt-com'`.
  - `lastFixAt` обновлять при публикации любого водолаза.

  **Буи — оба режима:**

    ```
  - Симулированные буи появляются на карте так же, как реальные (через тот же `onBuoyUpdate` колбэк).
