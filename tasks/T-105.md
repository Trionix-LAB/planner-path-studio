# T-105: RWLT navigation system support (uNav RF Dongle)

**Issue:** #105
**Branch:** `feat/105-rwlt-unav-com`
**Spec:** New requirement — добавить R-XXX в `spec/spec.md` раздел «Оборудование»

---

## Why

Сейчас приложение поддерживает Zima2R, GNSS UDP и GNSS COM как источники
навигации. Система RWLT (uNav RWLT RF Dongle) — гидроакустический трекер
реального времени для подводных объектов (водолазы, ТНПА). Донгл
подключается по COM-порту и выдаёт NMEA0183 + проприетарные сообщения
UNV/RWL. Добавление поддержки позволит отображать позиции водолазов
(режим RedPhone-DX) и пингера в реальном времени без сторонних программ.

## Протокол uNav (источник истины)

**Физический уровень:**
```
Baudrate: 38400, Data bits: 8, Stop bits: 1, Parity: No, Flow control: No
```

**Контрольная сумма:** XOR всех байт между `$` и `*`, hex 2 символа.
Формула та же, что в `validateChecksum` в `gnss-udp/protocol.ts` — можно переиспользовать.

**Входящие сообщения от донгла:**

| Сообщение | Режим | Поля | Применение |
|---|---|---|---|
| `$GNGGA,...*hh` | pinger | поля [2..5]=lat/lon, [6]=fixType, [8]=HDOP (радиальная ошибка), [9]=depth (со знаком «-») | Позиция пингера + глубина |
| `$GNRMC,...*hh` | pinger | поля [3..6]=lat/lon, [8]=course | Позиция + курс пингера |
| `$PUWV3,tID,tLat,tLon,tDpt,tCrs,tRer,Age*hh` | divers | tID=агент, tLat/tLon=координаты, tDpt=глубина, tCrs=курс, tRer=радиальная ошибка, Age=секунды | Позиция водолаза |
| `$PUWV4,tID,rpLt,rpLn,dst2rp,crs2rp,crs4rp,Age*hh` | both | Параметры до опорной точки | Не нужно для MVP, пропустить |
| `$PUWV5,gnssLt,gnssLn,gnssCrs,gnssSog*hh` | both | GNSS встроенного приёмника донгла | Не нужно для MVP, пропустить |
| `$PUWV6,dataID,dataValue*hh` | both | Доп. телеметрия пингера | Не нужно для MVP, пропустить |
| `$PRWLA,bID,bLt,bLn,bDpt,bBat,pData,bTOA,bMSR*hh` | both | Данные от буя: ID(1-4), lat, lon, глубина антенны, напряжение, TOA, MSR | Не нужно для MVP, пропустить |

**Глубина из GGA:** поле Altitude (индекс 9) передаётся со знаком «-» (отрицательное = под водой).
Нормализация: `depth = altitude < 0 ? -altitude : 0`.

**Режимы работы (`rwlt_mode`):**
- `0` (pinger): позиция обновляется из GGA/RMC каждые ~2 секунды. Один агент = пингер.
- `1` (divers): позиция каждого водолаза из PUWV3.tID → agentId. Обновление — после каждой PTT-передачи (нерегулярно).

  **Маппинг tID → agentId** (режим divers): **фиксированная таблица** — использовать `missionDivers` из состояния миссии.
  Найти `DiverUiConfig` где `beacon_id === String(tID)`, взять его `uid` как `entity_id`.
  Если совпадения нет → пропустить пакет (не публиковать fix для неизвестного tID).

### Шаг 2: Парсер протокола
- [ ] Создать `src/features/devices/rwlt-com/protocol.ts`.

  **Типы:**
  ```ts
  export type RwltMessageKind = 'GGA' | 'RMC' | 'PUWV3' | 'UNKNOWN';

  export type RwltGgaMessage = {
    kind: 'GGA';
    raw: string;
    lat: number;
    lon: number;
    depthM: number;       // нормализовано: -altitude (≥ 0)
    radialErrorM: number | null; // HDOP поле
    hasFix: boolean;
  };

  export type RwltRmcMessage = {
    kind: 'RMC';
    raw: string;
    lat: number;
    lon: number;
    courseDeg: number | null;
    hasFix: boolean;
  };

  export type RwltPuwv3Message = {
    kind: 'PUWV3';
    raw: string;
    targetId: number;      // tID
    lat: number;
    lon: number;
    depthM: number;
    courseDeg: number | null;
    radialErrorM: number | null;
    ageSeconds: number | null;
  };

  export type UnknownRwltMessage = { kind: 'UNKNOWN'; raw: string };

  export type ParsedRwltMessage =
    | RwltGgaMessage | RwltRmcMessage | RwltPuwv3Message | UnknownRwltMessage;
  ```

  **Функции:**
  - `validateNmeaChecksum(line: string): string | null` — XOR checksum, вернуть payload или null.
    Можно скопировать из `gnss-udp/protocol.ts` (`validateChecksum`) или переиспользовать.
  - `parseCoordinate(coord: string, hemi: string, isLat: boolean): number | null` — ddmm.mmmm → decimal degrees.
    Идентична функции в `gnss-udp/protocol.ts` — скопировать/переиспользовать.
  - `parseRwltLine(line: string): ParsedRwltMessage` — главная точка входа.
  - `splitRwltDatagram(datagram: string): string[]` — split по `\r?\n`, trim, filter empty.

  **Логика parseRwltLine:**
  1. Валидировать checksum. Если невалидна → `UNKNOWN`.
  2. Разбить на поля по `,`.
  3. По первому полю (sentenceId):
     - Оканчивается на `GGA` → parse GGA:
       - lat = parseCoordinate(fields[2], fields[3], true)
       - lon = parseCoordinate(fields[4], fields[5], false)
       - fixQuality = parseNumber(fields[6]); hasFix = fixQuality > 0 && lat != null && lon != null && !(lat==0 && lon==0)
       - altitude = parseNumber(fields[9]); depthM = altitude !== null && altitude < 0 ? -altitude : 0
       - radialErrorM = parseNumber(fields[8])
     - Оканчивается на `RMC` → parse RMC:
       - status = fields[2]; hasFix = status === 'A'
       - lat, lon через parseCoordinate(fields[3], fields[4]) / (fields[5], fields[6])
       - courseDeg = parseNumber(fields[8])
     - `PUNV3` или `PUWV3` (первое поле может быть `PUNV` + id=`3` или сразу `PUWV3`) — проверить реальный формат по документации: сообщение `$PUWV3,...` → sentenceId = `PUNV3` после strip `$P`.
       Парсить: fields[1]=tID(int), fields[2]/fields[3]=lat/lon (decimal, не ddmm!), fields[4]=tDpt, fields[5]=tCrs, fields[6]=tRer, fields[7]=Age
     - Иначе → `UNKNOWN`.

  > **Важно:** Координаты в PUWV3 передаются как decimal degrees (не ddmm.mmmm как в GGA/RMC).
